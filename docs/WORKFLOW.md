# 工作流程

## 开发迭代流程

每次功能迭代遵循以下流程：

1. **实现功能**
   - 在对应层级实现功能
   - 遵循架构原则（RNG 注入、分层等）

2. **补充测试**
   - 为新功能编写测试
   - 确保测试覆盖核心逻辑

3. **更新文档**
   - 如有架构变更，更新 `ARCHITECTURE.md`
   - 如有新功能，更新 `ROADMAP.md`

4. **运行测试**
   - 执行 `npm test` 确保所有测试通过（全绿）

5. **提交代码**
   - 通过测试后提交代码
   - 使用清晰的 commit message

## 测试命令

- `npm test`: 运行所有测试（单次）
- `npm run test:watch`: 监视模式运行测试
- `npm run test:ui`: 打开测试 UI
- `npm run test:coverage`: 生成覆盖率报告
